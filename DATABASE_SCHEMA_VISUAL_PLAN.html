XMETA Pay Visual Database Schema Plan

A browser-viewable version of the full practical MVP database plan for the admin/school dashboard and parent portal. This file is documentation only and does not change the live SQL import file.

MySQL / XAMPP InnoDB utf8mb4 Admin + Parent Portals

Project Overview

Admin/school portal School setup, student records, parent directory, tuition, collections, allowance, store transactions, and reports.
Parent portal Registration, login, linked students, enrollment, fee viewing, tuition payment, receipts, history, and wallet top-up.
Auth foundation Keep the existing shared users table with role-specific admin_profiles and parent_profiles.

Core Table Groups

Module Tables Main purpose
Auth users, admin_profiles, parent_profiles Login, registration, role redirects, and account status.
School setup schools, school_years, grade_levels, sections School structure used by admin dashboards and student placement.
Students students, student_guardians Student profiles and parent-to-student links.
Enrollment enrollments, enrollment_documents Enrollment status, grade/section assignment, and required documents.
Fees fee_types, student_fee_assignments Tuition, other fees, due dates, and balances.
Payments payments, payment_allocations, receipts Payment records, fee allocation, and receipt generation.
Wallet wallets, wallet_transactions Student allowance balance, top-ups, purchases, and adjustments.
Store store_merchants, store_transactions Canteen and school store spending logs.
Notifications notification_logs Payment reminders, receipts, low-wallet alerts, and enrollment updates.

  USERS {
    bigint id PK
    enum role
    varchar name
    varchar email
    varchar phone
    varchar password_hash
    enum status
  }

  ADMIN_PROFILES {
    bigint id PK
    bigint user_id FK
    varchar school_name
    enum staff_role
  }

  PARENT_PROFILES {
    bigint id PK
    bigint user_id FK
    varchar student_name
    varchar student_reference
    enum relationship
  }

  SCHOOLS {
    bigint id PK
    varchar name
    varchar code
    enum status
  }

    bigint id PK

    varchar name

    enum status

    bigint school_id FK
    varchar name

  SECTIONS {
    bigint id PK
    bigint school_id FK

    bigint grade_level_id FK
    varchar name
  }

  STUDENTS {

    bigint school_id FK

  STUDENT_GUARDIANS {
    bigint id PK

    bigint student_id FK

    bigint grade_level_id FK
    bigint section_id FK
    enum status

  ENROLLMENT_DOCUMENTS {
    bigint id PK

flowchart TD
  A["Admin opens students page"] --> B["Create or search student by reference"]

Suggested Implementation Order

  1. Keep the current auth schema working first.
  2. Add school setup tables.
  3. Add student and guardian tables.
  4. Add enrollment tables.
  5. Add fee tables.
  6. Add payment and receipt tables.
  7. Add wallet and wallet transaction tables.
  8. Add store merchant and store transaction tables.
  9. Add notification logs when reminders are implemented.
  10. Build reports from operational query views instead of separate report storage tables.
